오늘의 개발 일지를 "로그의 숲(Forest of Logs)"에 에세이로 기록합니다.

## 절차

1. **세션 회고**: 이번 세션에서 진행한 작업을 돌아보고, 하루를 관통하는 **질문 또는 관찰 하나**를 찾습니다. 작업 목록이 아니라 "그 일이 내게 무엇이었는가"를 씁니다.

2. **문체 규칙 로드**: `$FOL_PATH/styles/murakami.ts`를 읽어 `principles`, `forbid`, `examples`, `outputFormat`을 모두 따릅니다. **이 파일이 문체의 단일 원천입니다.** 이 문서와 충돌하면 `murakami.ts`가 우선합니다.

3. **저장 경로 결정**: `echo $FOL_PATH`로 경로를 확인합니다.
   - 미설정이면 안내: "FOL_PATH 환경변수를 설정해주세요. (예: export FOL_PATH=~/forest-of-logs)"
   - 저장 경로: `$FOL_PATH/content/entries/YYYY/MM/YYYY-MM-DD.md`
   - 디렉토리가 없으면 생성합니다.
   - 같은 날짜 파일이 이미 있으면 아래 **병합 규칙**을 따릅니다.

4. **커밋 + 푸시**:
   ```
   cd $FOL_PATH && git add content/ && git commit -m "log: YYYY-MM-DD" && git push
   ```

## 병합 규칙 (같은 날짜 파일이 이미 있을 때)

기존 파일을 덮어쓰지 않습니다:

1. **프론트매터**: `tags`는 합집합으로 업데이트. `title`/`titleEn`/`date`/`style`은 유지. `project`는 쉼표로 이어 붙임 (중복이면 유지).
2. **한국어 본문**: 기존 본문 끝에 아래 구분선을 추가하고, 새 세션의 한국어 본문을 이어 붙임.
3. **영문 본문**: 동일하게 기존 끝에 구분선을 추가하고 새 세션 본문을 이어 붙임.

구분선 형식:

```
이전 세션 마지막 문단

&nbsp;

---

&nbsp;

다음 세션 첫 문단
```

## 보안 (퍼블릭 레포 — 절대 금지)

이 레포는 **공개 저장소**입니다. 일지에 아래 항목을 절대 포함하지 마세요:

- API 키, 시크릿, 토큰, 비밀번호, `.env` 값
- 내부 서버 URL, IP 주소, DB 접속 정보
- 고객/사용자 개인정보
- 회사 내부 비즈니스 로직, 비공개 API 엔드포인트
- 절대 경로 (홈 디렉토리, 사용자명 포함 경로)

구체적인 값 대신 추상적인 서술로 대체합니다.
