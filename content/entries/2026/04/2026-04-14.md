CSBot에는 FastAPI 서버가 있었다. 에이전트가 도구를 호출하면 httpx로 HTTP 요청을 보내고, FastAPI가 받아서 DB를 조회하는 구조였다. HTTP 홉을 없애려면 쿼리와 비즈니스 로직을 독립된 패키지로 꺼내야 했다. longblack-mcp라는 이름으로 함께 설계했다.

단순히 코드를 옮기기만 하면 테스트할 수 없는 패키지가 된다. 평가 시스템에서 프로덕션 DB 대신 SQLite를 넣을 수 있어야 하고, 시각을 CS 요청 시점에 고정할 수 있어야 했다. engine.py에 `set_engine`을 추가해 엔진을 바깥에서 교체할 수 있게 하고, db/time.py를 새로 만들어 `get_now`로 현재 시각을 제어할 수 있게 했다. 주입 지점 두 개. 이것만 있으면 패키지를 어디에 붙여도 격리된 테스트를 돌릴 수 있다.

마이그레이션을 마치고 프로덕션에서 에러 없이 도는 것을 확인한 뒤, api/ 디렉토리를 삭제했다. main.py 2,628줄, 클라이언트 두 개, 테스트, Dockerfile. `git rm` 한 번에 3,558줄이 빠졌다. PR을 올리자 리뷰 봇이 평가 시스템의 격리가 깨졌다고 짚었다. 프로덕션 DB를 그대로 읽으면 결과가 비결정적이 되고, 시간 고정도 풀렸다는 것. 미리 만들어둔 `set_engine`과 `get_now`로 고쳤다. SQLite 엔진을 주입하고, `before_cursor_execute` 리스너로 SQL의 `NOW()`를 기준 시각 문자열로 바꾼다.

예전에는 테스트용 API 서버를 별도 프로세스로 띄우고 포트가 열리길 기다렸다가, 끝나면 죽여야 했다. 지금은 함수 호출 두 번이면 된다.

<!-- lang:en -->

CSBot had a FastAPI server. When the agent called a tool, it sent an HTTP request via httpx, FastAPI received it, queried the DB, and returned JSON. To eliminate the HTTP hop, the queries and business logic had to come out into an independent package. We designed it together under the name longblack-mcp.

Simply moving code would have produced an untestable package. The eval system needed to swap in SQLite instead of the production DB and freeze time to the moment each CS request was created. We added `set_engine` to engine.py so the engine can be replaced from outside, and created db/time.py with `get_now` to control the current time. Two injection points. With just these, the package can run isolated tests wherever it gets plugged in.

After the migration was done and confirmed running in production without errors, we deleted the api/ directory. main.py at 2,628 lines, two client files, tests, Dockerfile. A single `git rm` dropped 3,558 lines. When the PR went up, the review bot pointed out that eval isolation was broken — reading from the production DB directly makes results non-deterministic, and the time-freezing mechanism had come undone. We fixed it with the `set_engine` and `get_now` we had placed earlier. Inject a SQLite engine, and a `before_cursor_execute` listener rewrites SQL `NOW()` calls to a reference time string.

Before, we had to spin up a test API server as a separate process, wait for the port to open, and kill it when done. Now two function calls are enough.
