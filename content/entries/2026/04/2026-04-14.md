longblack-mcp라는 패키지를 함께 설계하고, 그것을 CSBot에 적용하는 작업을 했다. CSBot의 FastAPI 서버 안에 갇혀 있던 DB 쿼리와 비즈니스 로직을 독립된 패키지로 꺼내는 것이 핵심이었다. 에이전트가 HTTP를 거치지 않고 핸들러를 직접 호출할 수 있게 되면, 중간에 놓인 FastAPI 서버는 더 이상 필요하지 않다. 그 결과로 2서비스 구조가 1서비스로 줄었고, 3,500줄이 넘는 API 코드를 삭제할 수 있었다.

longblack-mcp를 단순한 코드 추출이 아니라 제대로 된 공유 라이브러리로 만들려면 테스트 격리를 지원해야 했다. `set_engine`으로 DB 엔진을 외부에서 교체할 수 있게 하고, `get_now`로 현재 시각을 제어할 수 있게 했다. 이 두 개의 주입 지점이 있어야 평가 시스템에서 프로덕션 DB 대신 SQLite를 넣고, 시간을 CS 요청 시점에 고정할 수 있다.

평가 시스템의 재설계는 PR 리뷰 봇과의 대화에서 구체화됐다. 봇이 두 가지를 정확히 짚었다. 프로덕션 DB를 직접 읽으면 결과가 비결정적이 된다는 것, 시간 고정 장치가 깨졌다는 것. longblack-mcp에 만들어둔 주입 지점 덕분에 해결은 깔끔했다. 평가 스크립트에서 SQLite 엔진을 만들어 `set_engine`으로 넣고, SQL의 `NOW()` 함수를 이벤트 리스너로 가로채서 기준 시각으로 치환한다. 예전에 별도 프로세스로 돌리던 테스트 API 서버가 하던 일을 같은 프로세스 안에서 더 간결하게 할 수 있게 된 것이다.

공유 라이브러리가 자리를 잡으면, 그 위에 얹혀 있던 것들이 단순해진다. CSBot은 API 서버 없이 에이전트 하나로 돌아가고, 평가 시스템은 프로세스 하나에서 격리된 테스트를 실행한다. 삭제한 코드보다 추가한 코드가 훨씬 적다는 사실이, 방향이 맞았다는 가장 솔직한 증거다.

We designed a package called longblack-mcp together and applied it to CSBot. The core idea was extracting the DB queries and business logic trapped inside CSBot's FastAPI server into an independent package. Once the agent can call handlers directly without going through HTTP, the FastAPI server sitting in between becomes unnecessary. As a result, a two-service architecture shrank to one, and over 3,500 lines of API code could be deleted.

To make longblack-mcp a proper shared library rather than a mere code extraction, it needed to support test isolation. We added `set_engine` so the DB engine can be swapped from outside, and `get_now` so the current time can be controlled. These two injection points are what allow the eval system to use SQLite instead of the production DB and freeze time to the CS request timestamp.

The eval system redesign took shape through conversation with the PR review bot. It pinpointed two things exactly. Reading from the production DB makes results non-deterministic. The time-freezing mechanism was broken. Thanks to the injection points we had built into longblack-mcp, the fix was clean. The eval script creates a SQLite engine and injects it via `set_engine`, then intercepts SQL `NOW()` calls with an event listener and rewrites them to reference time strings. What a separate test API server process used to do can now be done more concisely within the same process.

When a shared library finds its place, everything built on top of it becomes simpler. CSBot runs as a single agent without an API server, and the eval system executes isolated tests in a single process. The fact that far less code was added than deleted is the most honest evidence that the direction was right.
